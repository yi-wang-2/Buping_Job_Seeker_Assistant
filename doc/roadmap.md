## 项目定位

- 目标用户：应届生为主，兼顾社招。
- 核心定位：AI 智能助手，而不是单纯工具。
- 差异化策略：通过 AI 工程深度建立技术壁垒。
- 技术博客输出：每个深度功能可沉淀成一篇技术博客。

## 阶段一：AI 体验优化

### 1. AI 智能改写助手

目标：编辑器选中文本，一键 AI 改写。

改写模式：

- 更量化：数字和数据。
- 更专业：行业术语。
- 更简洁：去除冗余。
- 修正语法。

后续增强：

- 流式响应。
- Diff 预览。
- 一键应用 / 取消。

### 2. 模拟面试体验升级

优化点：

- 打字机效果。
- 上下文管理。
- 每轮实时评估。
- 可选语音输入和转写。

## 阶段二：AI 工程深度

### 3. 上下文管理系统

目标：统一管理短期记忆、长期记忆、工作记忆和 token 预算。

建议模块：

- `src/libs/ai_engine/context_manager.py`
- 多种压缩策略。
- Token 预算分配器。
- 关键信息提取。

### 4. Skills 系统

将 AI 能力模块化：

- `resume_writer`
- `mock_interviewer`
- `interview_coach`
- `text_rewriter`
- `jd_analyzer`
- `skill_matcher`
- `career_advisor`

每个 Skill 包含：

- System Prompt。
- Tools。
- Examples。
- Token Budget。
- Metadata。

### 5. Token 优化引擎

目标：保持质量前提下降低 token 消耗。

方向：

- Prompt caching。
- 增量更新。
- 智能截断。
- Token 预算分配。

### 6. 长期记忆系统

场景：

- 记录用户求职偏好。
- 记录简历风格偏好。
- 归档历史 JD。
- 跟踪简历迭代历史。

建议存储：本地 SQLite，隐私优先。

## 阶段三：进阶 AI 能力

- RAG 知识库：简历范文、公司面经、行业技能图谱。
- Multi-Agent 协作：简历分析师、JD 匹配师、模拟面试官、职业规划师。
- 全链路流式响应和可取消任务。

## 阶段四：求职闭环

- 申请跟踪看板。
- 面试时间提醒。
- 薪资对比。
- AI 从邮件提取面试信息。
- 求职仪表盘和 AI 洞察。
